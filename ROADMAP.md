# 3D 知识图谱可视化系统 — 架构路线图

## 整体架构

```
                         ┌─────────────────────┐
                         │      核心 KG          │
                         │  (exercises-dataset   │
                         │   → Neo4j/NetworkX)   │
                         └──────────┬────────────┘
                                    │
              ┌─────────────────────┼─────────────────────┐
              │                     │                     │
    ┌─────────▼─────────┐ ┌────────▼────────┐  ┌─────────▼─────────┐
    │    路径 1          │ │    路径 2        │  │    路径 3          │
    │  多Agent辩论问答   │ │  KG可视化探索     │  │  文档→KG自动构建   │
    └───────────────────┘ └──────────────────┘  └───────────────────┘
              │                     │                     │
              └─────────────────────┼─────────────────────┘
                                    │
                          ┌─────────┴─────────┐
                          │  共享基础设施       │
                          │  LLM API / Neo4j   │
                          │  前端 / 后端        │
                          └───────────────────┘
```

每条路径是独立闭环，共享一个 KG 核心。新路径不破坏旧路径。

## 路径说明

### 路径 2：KG 可视化探索 ✅ 已完成

- 3D Three.js 知识图谱可视化
- Liquid glass 节点材质 + 发光连线
- 点击聚焦 → BFS 径向环形布局
- 聚焦扩散暗度（离聚焦节点越远越暗）
- 聚焦高亮光环
- 拖动节点 + 松手回弹
- 悬停高亮（debounce 150ms）
- CSS2D 文字标签
- OrbitControls 旋转/缩放/平移
- 文件：`index3d-bounce.html`

### 路径 1：多 Agent 辩论问答 🔲 待开始

- 用户输入自然语言问题
- 多个 Agent 从不同视角检索 KG + 推理
  - 检索 Agent（查 KG 找相关实体）
  - 推理 Agent（基于证据链推导结论）
  - 验证 Agent（交叉验证其他 Agent 的结论）
  - 调度 Agent（协调辩论流程）
- Agent 之间互相辩论、交叉验证
- 输出带推理路径和置信度的答案
- 推理过程在 3D 图谱上可视化展示
- 技术栈：LangChain/CrewAI + Neo4j + LLM API

### 路径 3：文档 → KG 自动构建 🔲 待开始

- 用户上传 PDF/Markdown 文档
- LLM 自动抽取实体和关系 → 三元组
- 自动存入 Neo4j 知识图谱
- 抽取结果在 3D 图谱上可视化展示
- 技术栈：LLM API（实体抽取）+ Neo4j + 前端上传

### 可扩展路径 🔲

- 路径 4：对比分析（选两个概念 → 并排对比差异）
- 路径 5：个性化推荐（用户画像 → 图谱推理 → 推荐）
- 路径 6：时间线追溯（知识演变时间轴 + 图联动）
- 路径 7：数据统计面板（KG 统计信息大屏可视化）

## 决策记录

1. **不做 CV/YOLO 相关**：设备限制 + 不想自己造数据集
2. **不训练模型**：只调 API，不微调
3. **不用 3d-force-graph 库**：保留自己的 liquid glass 视觉风格
4. **力模拟仅限拖动回弹**：不做全时力模拟（会散开）
5. **多路径架构**：横向扩展而非纵向加深，每条路径独立
6. **先设计再开发**：避免交互冲突（如弹簧 vs 旋转的前车之鉴）
